gitの使い方

gitでのローカルリポジトリのversion管理フロー
・ローカルリポジトリを作成(初期化)
・ワークツリーに変更があった場合に
  ステージに追加する。
・


1.ローカルリポジトリの作成

まずは、自分のPC内でファイルのバージョン管理するために
ローカルリポジトリを作成する。

git init
・リポジトリ
 ・圧縮ファイルが作成される。
.gitディレクトリが作成される
中身
  COMMIT_EDITMSG  HEAD  config  hooks  index  logs  'objects'  refs

2.変更をステージに追加する。

次にファイルを作成や変更があった場合にまずは、ステージに
現在のファイルの状況を追加する。
#ファイル名を指定する場合
git add <ファイル名>
＃ディレクトリを指定する場合
git add <ファイル名>
#ワークツリー内の全てのファイルを指定する場合
git add .

上記をする事でリポジトリ内に圧縮ファイルが作成されステージ内のインデックスにてファイル名と圧縮ファイルがマッピングされる。
なお、ファイルの変更があると圧縮ファイルの中身が変わるので、新たにインデックスに追加をされる。


3.変更記憶する

git commit
git commit -m "メッセージ"
